## 二、产品定位重申

### 2.1 网页端（Web）
- **核心能力**：图片目标检测
- **辅助功能**：历史记录、AI问答、知识库、个人中心
- **不做的功能**：视频检测（上传视频文件检测）、实时视频流检测

### 2.2 未来移动端App（规划中）
- **核心能力**：实时视频检测（调用摄像头）
- **辅助功能**：图片检测、历史同步、推送通知
- **时间规划**：网页端稳定后启动

## 四、本次任务：网页端移除视频检测功能

### 4.1 任务目标
从网页端（前端+后端）移除所有视频检测相关的UI和接口，明确网页端仅支持图片识别。

### 4.2 前端修改范围

| 文件 | 修改内容 |
|------|----------|
| `views/Detect.vue` | 移除"视频检测"Tab，只保留图片检测；移除视频上传相关逻辑 |
| `views/History.vue` | 移除"视频"筛选按钮，只保留全部/图片 |
| `views/HistoryDetail.vue` | 移除视频播放相关逻辑，只支持图片详情 |
| `views/Profile.vue` | 移除"视频检测"统计卡片，只保留总检测、今日、图片 |
| `views/admin/Detections.vue` | 移除类型筛选中的"视频"选项 |
| `api/detection.js` | 移除 `detectVideo` 接口 |
| 其他文件 | 检查并移除视频相关的import、变量、函数 |

### 4.3 后端修改范围

| 文件 | 修改内容 |
|------|----------|
| `routes/detection.py` | 移除 `/api/detect/video` 接口 |
| `services/yolo_service.py` | 保留 `detect_video` 方法（供未来移动端复用），但路由层不暴露 |
| `config.py` | 保留 `ALLOWED_VIDEO_EXTENSIONS` 配置（代码层面保留，不删除） |
| `routes/history.py` | 历史记录查询保留type字段支持（数据库层兼容） |

**注意**：后端代码层面对视频检测的支持尽量保留（service层、model层），只是不在API层暴露给网页端。这样未来移动端App开发时可以复用后端逻辑。

### 4.4 验收标准

1. 检测页面只有"图片检测"，没有"视频检测"Tab
2. 历史记录筛选只有"全部"和"图片"，没有"视频"
3. 个人中心统计只有总检测、今日检测、图片检测，没有视频检测
4. 管理后台检测记录筛选没有"视频"选项
5. 后端 `/api/detect/video` 接口不可访问（或返回404）
6. 所有现有图片检测功能正常，不受影响
7. 数据库表结构不变，历史视频数据保留（不删数据）

## 六、任务执行工作流程

根据项目规范中的任务协作流程：

```
管理任务（当前）
    ↓
输出任务文档 + 提示词（本文档）
    ↓
用户将提示词分发给：前端开发Task + 后端开发Task
    ↓
两个开发Task分别执行代码修改
    ↓
管理Task验证执行结果
    ↓
通过 → 进入下一阶段开发
不通过 → 返回开发Task修改
```

### 关键词触发方式
- `执行前端任务 0702-精简` → 前端开发Task读取本文档，执行5.1节的前端提示词
- `执行后端任务 0702-精简` → 后端开发Task读取本文档，执行5.2节的后端提示词
