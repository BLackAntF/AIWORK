# 知识库与AI问答功能优化任务（0702-知识库）

**任务编号**：0702-知识库  
**任务类型**：功能优化与体验提升  
**依赖条件**：无需真实 LLM API / 向量数据库，基于现有 Mock 模式优化

## 一、任务背景

## 四、详细需求说明

### 4.1 打字机效果
- AI 回答返回后，逐字显示（速度约 30-50ms/字）
- 显示过程中可显示"停止"按钮（可选，本次不做停止功能，只有展示）
- 打字完成后显示完整内容和操作按钮

### 4.2 Markdown 渲染
- 使用 `marked` 或 `markdown-it` 库（检查项目是否已有，没有则安装）
- 支持：标题、加粗、斜体、列表、代码块、引用、链接
- 代码块需要语法高亮（可选，本次可先不用高亮）
- 引用来源区域保持原样式，不参与 Markdown 渲染

### 4.3 复制回答
- 每条 AI 回答气泡右上角显示"复制"图标按钮
- 点击后复制纯文本内容到剪贴板
- 复制成功显示"已复制"提示（ElMessage）

### 4.4 重新生成
- 仅对最后一条 AI 回答显示"重新生成"按钮
- 点击后删除最后一条AI回答，重新发送相同问题
- 保留 Mock 模式的随机性（每次重新生成回答可能不同）

### 4.5 常见问题快捷入口
- 新对话（无消息）时，在聊天区域显示常见问题卡片
- 示例问题：
  - "番茄叶斑病怎么防治？"
  - "如何识别植物病害？"
  - "常见病虫害有哪些？"
  - "病害防治的基本原则是什么？"
- 点击问题卡片直接发送该问题

### 4.6 知识详情预览弹窗
- 管理后台知识列表每行增加"查看"按钮
- 点击后弹出 Dialog，显示知识完整内容
- 内容支持 Markdown 渲染
- 显示标题、分类、来源、创建时间、状态等元信息

### 4.7 Markdown 编辑预览
- 新增/编辑知识对话框改为左右分栏布局
- 左侧：Markdown 文本编辑器（textarea）
- 右侧：实时预览（Markdown 渲染结果）
- 窗口宽度不足时改为上下布局

### 4.8 会话标题自动生成
- 后端保存第一条用户消息时，自动截取前 20 个字符作为会话标题
- 会话列表接口返回会话标题
- 已有会话如果没有标题，使用第一条消息的前20字

## 五、开发提示词

### 5.1 前端开发提示词

**任务**：知识库与AI问答功能前端优化

**读取文件**：
- `d:\Develop\CODE\AIWORK\Projects\yolo-detection-system\frontend\src\views\Knowledge.vue`
- `d:\Develop\CODE\AIWORK\Projects\yolo-detection-system\frontend\src\views\admin\Knowledge.vue`
- `d:\Develop\CODE\AIWORK\Projects\yolo-detection-system\frontend\src\api\knowledge.js`

**完成以下工作**：

#### 用户端问答优化（Knowledge.vue）
1. **Markdown 渲染**：安装 `marked` 库（如果没有），将 AI 回答内容从纯文本改为 Markdown 渲染。引用来源区域保持原样。
2. **打字机效果**：AI 回答返回后，逐字显示内容（30ms/字）。使用 `displayContent` 字段存储当前显示内容，打字过程中不显示操作按钮，完成后显示。
3. **复制回答**：每条 AI 回答气泡右上角添加复制按钮，点击复制回答纯文本到剪贴板，成功后 ElMessage 提示。
4. **重新生成**：在最后一条 AI 回答下方添加"重新生成"按钮，点击后删除最后一条回答，重新调用提问接口。
5. **常见问题**：新对话（messages 为空）时，在聊天区域显示 4 个常见问题卡片，点击直接发送该问题。常见问题列表：
   - 番茄叶斑病怎么防治？
   - 如何识别植物病害？
   - 常见病虫害有哪些？
   - 病害防治的基本原则是什么？
6. **思考中状态**：调用提问接口后、收到回答前，在消息列表末尾显示"正在思考..."的加载动画。

#### 管理后台优化（admin/Knowledge.vue）
1. **知识详情预览**：列表每行增加"查看"按钮，点击弹出 Dialog，显示知识完整内容（Markdown 渲染），包括标题、分类、来源、创建时间、状态。
2. **Markdown 编辑预览**：新增/编辑知识对话框改为左右分栏，左侧 textarea 编辑 Markdown，右侧实时预览。使用与用户端相同的 Markdown 渲染方式。
3. **导入结果反馈**：批量导入成功后，显示详细结果（成功X条），如果有失败也显示失败数量和原因。

**注意**：
- 保持现有玻璃拟态风格和暗色主题
- 所有交互使用 Element Plus 组件
- 打字机效果不要阻塞 UI
- Markdown 渲染要防止 XSS（使用 DOMPurify 或 marked 的 sanitize 选项）

**验收标准**：
1. AI 回答有打字机逐字显示效果
2. AI 回答支持 Markdown 格式渲染（加粗、列表、代码块等）
3. 每条 AI 回答有复制按钮，可复制内容
4. 最后一条 AI 回答有重新生成按钮
5. 新对话时显示常见问题快捷入口
6. 管理后台知识可查看详情（Markdown 渲染）
7. 新增/编辑知识有实时预览
8. 所有现有功能不受影响

### 5.2 后端开发提示词

**任务**：知识库与AI问答功能后端优化

**读取文件**：
- `d:\Develop\CODE\AIWORK\Projects\yolo-detection-system\backend\routes\knowledge.py`
- `d:\Develop\CODE\AIWORK\Projects\yolo-detection-system\backend\services\llm_service.py`
- `d:\Develop\CODE\AIWORK\Projects\yolo-detection-system\backend\services\knowledge_service.py`
- `d:\Develop\CODE\AIWORK\Projects\yolo-detection-system\backend\models.py`

**完成以下工作**：

1. **会话标题自动生成**：
   - 在 `/api/knowledge/ask` 接口中，如果是新会话（session_id 不存在或该 session 第一条消息），自动截取用户问题的前 20 个字符作为会话标题
   - 会话标题存储在 ChatHistory 的某个字段中（或新建 session 表，或在 metadata 中）。建议在 metadata 中添加 `session_title` 字段
   - `/api/knowledge/sessions` 接口返回会话列表时，优先使用 session_title，没有则用第一条消息前20字

2. **Mock 回答丰富化**：
   - 扩展 `llm_service.py` 中的 MOCK_ANSWERS，按问题类型分类：
     - 病害识别类（3-5个模板）
     - 防治方法类（3-5个模板）
     - 栽培技术类（3-5个模板）
     - 通用知识类（3-5个模板）
   - 根据问题关键词简单匹配，选择对应类型的模板
   - 没有匹配到类型时使用通用模板

3. **Mock 思考延迟**：
   - 在 `generate_answer` 方法的 mock 模式下，增加 1-2 秒随机延迟（使用 time.sleep）
   - 模拟真实 AI 处理时间

4. **提问限流**：
   - 在 `/api/knowledge/ask` 接口添加限流：同一用户每分钟最多 20 次提问
   - 使用内存中的简单计数器（用字典存储，key 为 user_id，value 为时间戳列表）
   - 超出限制返回 429 状态码和提示信息

**注意**：
- 保持现有代码风格和结构
- 限流使用内存缓存即可，不需要 Redis（开发环境）
- Mock 延迟不要影响其他接口性能
- 会话标题逻辑要兼容已有历史数据

**验收标准**：
1. 新会话自动生成标题（第一条消息前20字）
2. 会话列表接口返回标题字段
3. Mock 回答根据问题类型选择不同模板
4. AI 回答有 1-2 秒延迟
5. 同一用户每分钟提问超过 20 次返回 429
6. 所有现有接口功能不受影响
