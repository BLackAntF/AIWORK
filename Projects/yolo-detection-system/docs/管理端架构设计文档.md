# 系统管理端架构设计文档

## 一、架构概述

### 1.1 设计方案

采用**共用前后端、路由分离、权限控制**的架构方案：

| 方案 | 说明 |
|------|------|
| **前端** | 与用户端共用一套 Vue 项目，通过路由区分用户端和管理端 |
| **后端** | 与用户端共用 Flask 后端，新增管理端专用路由 |
| **权限** | 基于 JWT + 用户角色（role: user/admin），通过中间件控制访问 |
| **入口** | 管理端入口：`/admin` 路由，仅管理员可访问 |

### 1.2 架构图

```
┌─────────────────────────────────────────────────────────────┐
│                     Vue 3 前端应用                            │
├─────────────────────────────┬───────────────────────────────┤
│        用户端路由            │         管理端路由             │
│  /login, /detect,           │  /admin/*                     │
│  /knowledge, /history       │  （需 admin 角色）             │
├─────────────────────────────┴───────────────────────────────┤
│                   Axios + JWT Token                          │
├─────────────────────────────────────────────────────────────┤
│                     Flask 后端 API                            │
├─────────────────────────────┬───────────────────────────────┤
│      用户端 API 路由         │       管理端 API 路由          │
│  /api/auth, /api/detect,    │  /api/admin/*                 │
│  /api/knowledge, /api/history│  （需 admin 中间件）          │
├─────────────────────────────┴───────────────────────────────┤
│                   SQLite / MySQL 数据库                       │
└─────────────────────────────────────────────────────────────┘
```

## 二、权限设计

### 2.1 用户角色

| 角色 | 说明 | 权限范围 |
|------|------|---------|
| `user` | 普通用户 | 用户端全部功能（检测、问答、历史、个人中心） |
| `admin` | 管理员 | 用户端全部功能 + 管理端全部功能 |

### 2.2 权限中间件

后端已有 `login_required` 和 `admin_required` 中间件，管理端接口使用 `admin_required`：

```python
# middleware/auth_middleware.py（已有）
def login_required(fn):     # 验证登录
def admin_required(fn):     # 验证管理员权限
```

### 2.3 前端路由守卫

```javascript
// router/index.js
router.beforeEach((to, from, next) => {
  const userStore = useUserStore()
  
  // 管理端路由检查
  if (to.path.startsWith('/admin')) {
    if (!userStore.token) {
      next('/login')
    } else if (userStore.userInfo?.role !== 'admin') {
      next('/')  // 非管理员跳转首页
    } else {
      next()
    }
  }
  // ... 其他路由守卫逻辑
})
```

|------|------|------|------|

| role | string | 否 | 角色筛选：user/admin |
| status | string | 否 | 状态筛选：active/inactive |
| keyword | string | 否 | 用户名/邮箱搜索 |

**响应**：
```json
{

  "message": "success",
  "data": {
    "list": [

        "email": "test@example.com",
        "role": "user",
        "is_active": true,
        "created_at": "2024-01-01T00:00:00Z",
        "detection_count": 10,
        "chat_count": 5

    "total": 100,
    "page": 1,
    "page_size": 20
  }
}
```

## 八、技术要点

### 8.1 权限控制

- **后端**：所有管理端接口使用 `@admin_required` 装饰器
- **前端**：路由守卫检查 `userStore.userInfo.role === 'admin'`
- **菜单**：侧边栏根据角色显示用户端菜单或管理端菜单

### 8.2 数据一致性

- 删除用户时同时删除其检测历史、对话历史
- 删除知识分类时将该分类下的知识的 category 设为 null
- 切换模型时重新加载 YOLO 服务

### 8.3 批量操作

- 批量删除使用 DELETE 请求 + body: { ids: [] }
- 批量导入使用 multipart/form-data 上传文件
- 同步向量库时先清空再重新添加

### 8.4 统计数据缓存

- 统计接口可使用 Redis 缓存（5分钟过期）
- 导出报表直接查询数据库生成 CSV
