## 2. 总体设计

### 2.1 系统架构

系统采用**单层客户端直连架构**，移动端 APP 通过 BLE 蓝牙直接与智能加热炉设备通信，不经过服务器中转。整体架构如下：

```
┌─────────────────────────────────────────────────────────────┐
│                   移动终端 (手机 / Android / iOS)                 │
│  ┌─────────────────────────────────────────────────┐  │
│  │          智慧烹饪系统 APP (uni-app)              │  │
│  │  ┌─────────┐  ┌─────────┐  ┌──────────┐ │  │
│  │  │  首页    │  │ 控制页  │  │ 蓝牙管理   │ │  │
│  │  │ (index) │  │ (detail)│  │ (ble.js) │ │  │
│  │  └────┬────┘  └────┬────┘  └─────┬────┘ │  │
│  │       │              │               │        │        │  │
│  │       └──────────────┴───────────────┘        │  │
│  │                    │                         │  │
│  │           uni-app BLE API                       │  │
│  │                    │                         │  │
│  └────────────────────┼───────────────────────┘  │
│                       │                             │
└───────────────────────┼─────────────────────────────┘
                        │ BLE 蓝牙
                        │
┌───────────────────────┼─────────────────────────────┐
│              智能加热炉设备                              │
│  ┌──────────────────┐  ┌──────────────┐  ┌──────────┐   │
│  │ BLE 通信模块 │  │ 机芯控制器 │  │ 传感器  │   │
│  │ (FFB0服务)  │──│ (12KW)   │──│ (温度等)│   │
│  └──────────┬───────┘  └──────┬──────┘  └─────┬───┘   │
│         │                  │                    │       │
│         └──────────────────┴────────────────────┘       │
│                    电磁加热单元                              │
└─────────────────────────────────────────────────────┘
```

### 2.2 设计原则

- **模块化设计**：各功能模块独立封装，降低耦合度
- **单一职责**：每个模块只负责一个功能领域
- **可扩展性**：预留扩展接口，便于后续功能迭代
- **用户体验优先**：操作简洁直观，反馈及时明确
- **安全性**：异常告警及时，保护设备安全

### 2.3 技术选型

| 层次 | 技术选型 | 说明 |
|------|---------|------|
| 前端框架 | uni-app (Vue 2/3) | 跨平台，一套代码多端运行 |
| UI 风格 | 自定义组件 | 蓝白主题，卡片式设计 |
| 蓝牙通信 | uni-app BLE API | 系统原生蓝牙接口 |
| 通信协议 | ASCII 明文串口协议 | 12KW 机芯专用协议 |
| 数据存储 | uni.storage | 本地持久化存储 |
| 开发工具 | HBuilderX 3.x+ | uni-app 官方 IDE |

### 3.2 页面层模块设计

**职责**：应用首页，负责设备组展示、设备列表、蓝牙扫描添加、拖拽排序、测试预警入口等功能。

**子模块说明**：

| 子模块 | 功能描述 |
|--------|---------|
| 设备组管理 | 创建设备组、编辑组名、删除设备组、展开/收起 |

| 拖拽排序 | 长按拖拽、跨组移动、视觉反馈 |
| 批量控制 | 组内全部开机、全部关机 |
| 蓝牙扫描 | 初始化蓝牙、扫描设备、显示结果、添加设备 |
| 测试预警 | 模拟设备异常、触发报警跳转 |

#### 3.2.2 控制页模块 (detail.vue)

**职责**：设备详情控制页，提供单设备精细化控制功能。

| 指令日志 | 操作记录、设备应答、时间戳 |

### 3.3 工具层模块设计

#### 3.3.1 蓝牙通信模块 (ble.js)

**职责**：封装所有蓝牙相关操作，向上提供统一的调用接口。

**子模块说明**：

| 子模块 | 功能描述 |
|--------|---------|

## 4. 系统流程设计

### 4.1 蓝牙连接流程

```
用户点击添加设备
      │
      ▼
初始化蓝牙适配器
      │
      ├─ 成功 ──▶ 提示"请开启蓝牙"
      │             (失败
      ▼
开始扫描设备
      │
      ▼
发现设备 → 去重 → 添加到列表
      │
      ▼
5秒后自动停止
      │
      ▼
用户选择设备
      │
      ▼
创建BLE连接
      │
      ├─ 失败 ──▶ 提示连接失败
      │
      ▼
获取服务和特征值
      │
      ▼
开启通知监听
      │
      ▼
设备添加成功
```

### 4.2 指令发送与接收流程

```
用户操作
  │
  ▼
生成指令字符串 (如 >AF3
  │
  ▼
调用 bleManager 发送
  │
  ▼
BLE 写入特征值
  │
  ├─ 成功 ──▶ 写入日志
  │
  ▼
等待设备应答
  │
  ├─ >OK ──▶ 指令成功 ──▶ 更新状态
  │
  ├─ >AE* ──▶ 异常上报 ──▶ 触发报警
  │
  └─ >A... ──▶ 查询数据 ──▶ 解析显示
```

### 4.3 异常报警流程

```
设备上报异常码 (>AE1~6)
        │
        ▼
ble.js 消息监听捕获
        │
        ▼
解析异常码，获取描述
        │
        ▼
添加到报警列表
        │
        ├─────全局事件 (uni.$emit)
        │
        ▼
启动持续震动 (500ms间隔)
        │
        ▼
用户点击"关机"
        │
        ▼
发送 >AFF 关机指令
        │
        ▼
移除该条报警
        │
        ▼
无剩余报警？
  ├─ 是 ──▶ 停止震动
  └─ 否 ──▶ 继续震动
```

## 5. 接口设计

### 5.1 内部模块接口

#### 5.1.1 蓝牙管理对象 (bleManager)

| 方法名 | 参数 | 返回值 | 说明 |
|--------|------|--------|------|
| powerOn() | 无 | Promise | 发送开机指令 |
| powerOff() | 无 | Promise | 发送关机指令 |
| setGear(level) | level: Number(0-12) | Promise | 设置火力档位 |
| setZone(mode) | mode: String | Promise | 设置分区模式 |
| setInnerSegment(seg) | seg: Number(1-5) | Promise | 设置内区分段 |
| setOuterSegment(seg) | seg: Number(1-5) | Promise | 设置外区分段 |
| queryTemperature() | 无 | Promise | 查询温度 |
| queryPower() | 无 | Promise | 查询电参数 |
| onMessage(callback) | callback: Function | 无 | 注册消息监听 |
| isAckOk(data) | data: String | Boolean | 判断是否为成功应答 |
| isAlarm(data) | data: String | Boolean | 判断是否为异常码 |
| isQueryResponse(data) | data: String | Boolean | 判断是否为查询响应 |
| parseTemperatureData(data) | data: String | Object/null | 解析温度数据 |
| parsePowerData(data) | data: String | Object/null | 解析电参数 |

#### 5.1.2 页面间通信接口

| 事件名 | 参数 | 触发时机 |
|--------|------|---------|
| deviceAlarm | { deviceId, deviceName, alarmCmd, alarmLabel, timestamp | 设备异常上报时 |
| selectedDeviceId | (storage) | 当前选中设备标识 |
| pendingAlarm | (storage) | 待处理报警数据 |

### 5.2 蓝牙通信协议接口

详见《蓝牙通信协议文档 v2.0》。

#### 5.2.1 指令格式

```
格式：> + 指令标识 + [参数
开头：> (ASCII 0x3E)
结尾：无结尾符，指令总长固定为 4 字符（查询类为 4 字符）
```

#### 5.2.2 指令汇总

| 类别 | 指令 | 说明 | 应答 |
|------|------|------|------|
| 查询 | >AT? | 查询三路温度 | >A + 12位HEX |
| 查询 | >AP? | 查询电参数 | >A + 12位HEX |
| 开关机 | >AFO | 开机 | >OK |
| 开关机 | >AFF | 关机 | >OK |
| 档位 | >AF1~9 | 1~9档 | >OK |
| 档位 | >AFA/B/C | 10/11/12档 | >OK |
| 分区 | >ACI | 仅内区 | >OK |
| 分区 | >ACX | 仅外区 | >OK |
| 分区 | >ACL | 内外全区 | >OK |
| 内分段 | >AA1~5 | 内区1~5段 | >OK |
| 外分段 | >AB1~5 | 外区1~5段 | >OK |
| 异常 | >AE1~6 | 异常上报（设备主动） | 无 |

## 8. 出错处理设计

### 8.1 错误分类

| 错误类型 | 说明 | 处理方式 |
|---------|------|---------|
| 蓝牙未开启 | 用户未开启蓝牙 | 提示"请开启手机蓝牙" |
| 扫描失败 | 蓝牙扫描异常 | 显示错误信息，提供重新搜索 |
| 连接失败 | 设备连接不上 | 提示连接失败，重试 |
| 指令发送失败 | 写入特征值失败 | Toast提示"指令发送失败"（测试设备静默） |
| 设备离线 | 连接断开 | 禁用控制按钮，提示离线状态 |
| 异常上报 | 设备故障 | 报警卡片+震动提醒 |

### 8.2 容错机制

- **指令失败不阻塞**：单条指令失败不影响其他功能
- **状态同步**：UI 状态与设备状态尽量保持一致
- **数据本地持久化**：设备列表和分组信息本地保存
