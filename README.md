# 智慧烹饪系统

基于 uni-app 开发的智能烹饪设备蓝牙控制APP，支持蓝牙设备扫描连接、设备操控面板、异常报警等功能。

## 项目概述

本项目是智慧烹饪系统的第一期移动端APP，实现以下核心功能：

- **蓝牙设备管理**：扫描、连接/断开蓝牙烹饪设备
- **设备操控面板**：模拟真实设备的按键操作界面
- **异常报警**：接收设备异常上报并显示报警信息
- **指令日志**：记录所有收发指令和系统事件

## 技术栈

- **框架**：uni-app (Vue 3)
- **蓝牙**：uni-app BLE API
- **通信协议**：12KW 明文协议
- **打包平台**：Android / iOS / 微信小程序

## 快速开始

### 环境要求

- HBuilderX 3.x+
- Node.js 14+
- 支持蓝牙的 Android/iOS 设备或模拟器

### 运行

1. 使用 HBuilderX 打开项目
2. 选择运行平台（Android/iOS/微信开发者工具）
3. 点击运行

### 打包

- **Android**：发行 → 原生APP-云打包 → Android
- **iOS**：发行 → 原生APP-云打包 → iOS

## 蓝牙通信协议

### 设备UUID

| 类型 | UUID |
|------|------|
| 服务UUID | `0000FFB0-0000-1000-8000-00805F9B34FB` |
| 写特征值 | `0000FFB2-0000-1000-8000-00805F9B34FB` |
| 通知特征值 | `0000FFB1-0000-1000-8000-00805F9B34FB` |

### 12KW 指令集

#### 手机 → 设备

| 指令 | 说明 |
|------|------|
| `>OK` | 开机/关机/设置档位 |
| `>AT?` | 查询温度（3路） |
| `>AP?` | 查询电参数（电压/功率/电流） |

#### 设备 → 手机

| 指令 | 说明 |
|------|------|
| `>A` + 12位十六进制 | 温度上报（每4位=1路） |
| `>A` + 12位十六进制 | 电参数上报（电压+功率+电流） |
| `>AE1` | 机芯超温 |
| `>AE2` | 线盘超温 |
| `>AE3` | 过流保护 |
| `>AE4` | 电流异常 |
| `>AE5` | 锅具异常 |
| `>AE6` | 线盘测温开路 |

## 操控面板操作说明

设备连接后显示操控面板，模拟真实设备按键布局：

```
[定时] [减] [显示屏] [加] [开关]
```

### 操作流程

1. **双击开关** → 开机，显示屏显示 `- P0 -`
2. **点击"加"** → 从P0升至P1开始工作，继续点击升至P12
3. **点击"减"** → 降低档位，降至P0回到待机
4. **点击"定时"** → 进入定时设定模式，显示屏显示 `00:00`
5. 定时模式下按**加/减**调整时间（步长1分钟）
6. 再次按**定时**确认，开始倒计时，显示屏实时显示 `mm:ss` 剩余时间
7. 倒计时结束**自动关机**
8. **单击开关** → 手动关机


### 档位指示灯

显示屏下方有12个指示灯条，P1亮1个，P5亮5个，以此类推。

## 异常报警

设备上报异常时，APP会：
1. 在报警区域显示红色报警条目（异常码 + 描述 + 时间）
2. 弹出 Toast 提示

### 模拟报警（测试用）

无真实设备时，可点击操控面板底部的 **"🧪 模拟异常报警"** 按钮，选择异常类型进行测试。
