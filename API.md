## 1. 概述

本文档定义了智慧烹饪系统与智能加热设备之间的蓝牙 BLE 通信协议。设备通过低功耗蓝牙（Bluetooth Low Energy）与移动端应用进行数据交互。

### 1.1 协议特性

- **传输方式**：BLE GATT 协议
- **数据编码**：ASCII 字符串
- **指令格式**：以 `>` 开头，后跟指令标识符和可选参数
- **响应格式**：以 `>` 开头，后跟响应标识符和数据

|------|------|------|

## 2. 连接规范

### 2.1 UUID 定义

| 类型 | UUID | 说明 |
|------|------|------|
| 服务 UUID | `0000FFB0-0000-1000-8000-00805F9B34FB` | 烹饪设备服务 |
| 写特征值 | `0000FFB2-0000-1000-8000-00805F9B34FB` | 发送指令到设备 |
| 通知特征值 | `0000FFB1-0000-1000-8000-00805F9B34FB` | 接收设备返回数据 |

### 2.2 连接流程

```
移动端                         设备
  |                             |
  |-- 1. 扫描设备 ------------->|
  |<-- 2. 发现设备 -------------|
  |-- 3. 发起连接 ------------->|
  |<-- 4. 连接成功 -------------|
  |-- 5. 获取服务 ------------->|
  |<-- 6. 返回服务列表 ---------|
  |-- 7. 获取特征值 ----------->|
  |<-- 8. 返回特征值 -----------|
  |-- 9. 开启通知 ------------->|
  |<-- 10. 通知开启成功 --------|
  |                             |
  |     连接建立完成             |
```

### 2.3 连接参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 连接间隔 | 30-50ms | 动态调整 |
| 扫描超时 | 8s | 自动停止扫描 |
| 连接超时 | 10s | 连接失败重试 |
| MTU 大小 | 默认 23 bytes | 可协商至 512 bytes |

## 3. 指令集

### 3.1 基本控制指令

| 指令 | ASCII | 说明 | 参数 |
|------|-------|------|------|
| 开机 | `>A` | 设备上电启动 | 无 |
| 关机 | `>B` | 设备下电关闭 | 无 |
| 待机 | `>C` | 进入待机低功耗模式 | 无 |

### 3.2 模式选择指令

| 指令 | ASCII | 说明 | 参数 |
|------|-------|------|------|
| 煎炸模式 | `>D` | 切换至煎炸模式 | 无 |
| 炖煮模式 | `>E` | 切换至炖煮模式 | 无 |
| 蒸模式 | `>F` | 切换至蒸模式 | 无 |
| 烘焙模式 | `>G` | 切换至烘焙模式 | 无 |
| 保温模式 | `>H` | 切换至保温模式 | 无 |

### 3.3 温度控制指令

| 指令 | ASCII | 说明 | 参数 |
|------|-------|------|------|
| 升温 | `>I` | 增加设定温度 | 无 |
| 降温 | `>J` | 降低设定温度 | 无 |
| 设置温度 | `>K{temp}` | 设置目标温度 | temp: 0-250 整数 |

**示例**：
- `>K180` → 设置温度为 180°C
- `>K200` → 设置温度为 200°C

### 3.4 时间控制指令

| 指令 | ASCII | 说明 | 参数 |
|------|-------|------|------|
| 时间增加 | `>L` | 增加烹饪时间 | 无 |
| 时间减少 | `>M` | 减少烹饪时间 | 无 |
| 设置时间 | `>N{time}` | 设置烹饪时间(分钟) | time: 0-180 整数 |

**示例**：
- `>N30` → 设置时间为 30 分钟
- `>N120` → 设置时间为 120 分钟

### 3.5 功能控制指令

| 指令 | ASCII | 说明 | 参数 |
|------|-------|------|------|
| 开始烹饪 | `>O` | 启动烹饪程序 | 无 |
| 暂停 | `>P` | 暂停当前烹饪 | 无 |
| 停止 | `>Q` | 停止烹饪 | 无 |
| 定时 | `>R` | 设置定时启动 | 无 |

### 3.6 状态查询指令

| 指令 | ASCII | 说明 | 参数 |
|------|-------|------|------|
| 查询状态 | `>S` | 查询设备完整状态 | 无 |
| 查询温度 | `>T` | 查询当前温度 | 无 |
| 查询时间 | `>U` | 查询剩余时间 | 无 |

### 3.7 异常处理指令

| 指令 | ASCII | 说明 | 参数 |
|------|-------|------|------|
| 重置警报 | `>V` | 清除异常警报状态 | 无 |
| 紧急停止 | `>W` | 紧急停止所有操作 | 无 |

## 4. 数据格式

### 4.1 指令格式

```
> [指令标识符] [参数]
│  └── 1 字节 ASCII 字符
└── 起始标记 (0x3E)
```

### 4.2 设备响应格式

| 响应 | 格式 | 说明 |
|------|------|------|
| 温度上报 | `>T{value}` | 如 `>T185` 表示 185°C |
| 时间上报 | `>U{value}` | 如 `>U25` 表示剩余 25 分钟 |
| 状态上报 | `>S{state}` | 状态码见下表 |

### 4.3 设备状态码

| 状态码 | 说明 |
|--------|------|
| `0` | 待机中 |
| `1` | 预热中 |
| `2` | 烹饪中 |
| `3` | 已暂停 |
| `4` | 已完成 |
| `5` | 保温中 |
| `6` | 异常报警 |

### 4.4 异常码

| 异常码 | 说明 |
|--------|------|
| `E01` | 温度传感器故障 |
| `E02` | 加热元件异常 |
| `E03` | 通信超时 |
| `E04` | 电压异常 |
| `E05` | 过热保护触发 |

## 5. 通信流程

### 5.1 正常烹饪流程

```
移动端                         设备
  |                             |
  |-- 开机 (>A) -------------->|
  |-- 选择模式 (>D~>H) ------->|
  |-- 设置温度 (>K180) ------->|
  |-- 设置时间 (>N30) -------->|
  |-- 开始烹饪 (>O) ---------->|
  |                             |
  |<-- 温度上报 (>T{val}) -----|
  |<-- 时间上报 (>U{val}) -----|
  |                             |
  |-- 暂停 (>P) --------------->|
  |-- 继续 (>O) --------------->|
  |-- 停止 (>Q) --------------->|
  |-- 关机 (>B) --------------->|
```

### 5.2 异常处理流程

```
移动端                         设备
  |                             |
  |<-- 异常上报 (E01~E05) -----|
  |                             |
  |-- 查询状态 (>S) ---------->|
  |<-- 返回状态码 6 -----------|
  |                             |
  |-- 紧急停止 (>W) ---------->|
  |-- 重置警报 (>V) ---------->|
  |-- 查询状态 (>S) ---------->|
  |<-- 返回状态码 0 -----------|
```

## 6. 错误处理

### 6.1 通信超时

- 指令发送后 3 秒内未收到响应视为超时
- 超时后应重试 2 次
- 连续 3 次超时断开连接

### 6.2 数据校验

- 设备应验证指令格式是否正确
- 无效指令应忽略并返回错误码
- 温度/时间参数应在有效范围内

### 6.3 断线重连

- 检测到连接断开后自动尝试重连
- 重连间隔：1s, 2s, 4s, 8s（指数退避）
- 最大重连次数：5 次

## 7. 安全规范

### 7.1 连接安全

- 设备应验证连接来源
- 建议实现配对绑定机制
- 敏感操作需二次确认

### 7.2 指令安全

- 温度限制：0-250°C
- 时间限制：0-180 分钟
- 超范围参数自动裁剪

### 7.3 异常保护

- 超过安全温度自动断电
- 长时间无操作自动待机
- 硬件故障自动报警
