# 规范

约定俗成的做法、代码风格、命名规范、目录结构等。

## 代码质量

- 函数不超过 20 行（单一职责）
- 使用类型提示和 Docstring
- 禁止 Magic Value（用命名常量替代）
- 错误处理必须防御性编程
- 禁止无限制引入第三方库（建立白名单）

## 安全红线

- 禁止硬编码密钥、API Key、密码（使用环境变量）
- 所有数据库查询必须参数化
- 用户输入必须验证
- 禁止使用 `eval`、字符串拼接 SQL
- 敏感操作必须权限校验

## 工程流程

- AI 生成代码必须人工 review
- 必须有单元测试
- 禁止直接提交未检查的代码到主分支
- 分层级约束：Hard Constraints > Design Intent > Allowed Variance
