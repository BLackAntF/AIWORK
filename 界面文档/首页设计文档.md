## 1. 页面概述

### 1.1 页面定位

首页是APP启动后的默认页面，以**设备组**为核心组织方式，展示用户所有已连接的设备。用户可以通过设备组快速定位设备，点击设备卡片进入详情控制页。

### 1.2 核心功能

| 功能 | 说明 |
|------|------|
| 设备组展示 | 按用户创建的分组展示设备，默认收起 |
| 设备卡片 | 两列网格布局，显示图标、名称、火候档位、状态 |
| 创建设备组 | 右上角 [+] → 创建设备组 |
| 添加设备 | 右上角 [+] → 添加设备（蓝牙扫描或添加测试设备） |
| 编辑组名 | 点击组名直接编辑（"未分组设备"不可编辑） |
| 删除设备组 | 右上角 [+] → 删除设备组，组内设备移回"未分组设备" |
| 批量控制 | 设备组展开后显示"全部开机"/"全部关机"按钮 |
| 拖拽排序 | 长按1秒后拖拽卡片，可跨组移动设备 |
| 设备删除 | 点击卡片右上角红叉，未分组中删除，其他组中移回未分组 |
| 测试预警 | 底部"测试预警"区域，可模拟6种真实设备异常上报 |

### 1.3 页面状态

| 状态 | 说明 |
|------|------|
| 空状态 | 首次使用，无设备组、无设备，显示引导界面 |
| 有组无设备 | 已创建组但未添加设备，组内显示"此组暂无设备" |
| 正常状态 | 有设备组和设备，显示设备卡片网格 |

| 操作 | 效果 |
|------|------|
| 点击[全部开机] | 弹出确认框"确定开启组内所有在线设备？" → 确认后执行 |
| 点击[全部关机] | 弹出确认框"确定关闭组内所有在线设备？" → 确认后执行 |

### 6.5 默认分组

- 系统自动创建"未分组设备"分组，默认展开
- 新添加的设备自动归入"未分组设备"
- "未分组设备"不可删除
- "未分组设备"不可修改名称
- "未分组设备"**不显示**批量控制按钮（全部开机/全部关机）
- "未分组设备"**不显示** ⋮ 更多按钮（仅显示编辑组名选项）
- 用户可以将设备从"未分组"移动到其他组

### 6.6 组操作菜单（点击 ⋮ 按钮）

#### 非"未分组设备"

|------|------|
| 编辑组名 | 弹出输入框修改组名 |
| 删除组 | 确认后删除组，组内设备移回"未分组设备" |
| 取消 | 关闭菜单 |

## 8. 拖拽排序

### 8.1 拖拽流程

```
手指按下 → 悬停1秒（dragReady = true）→ 手指移动超过20px → 进入拖拽模式
  → 原始卡片变半透明（opacity: 0.3）
  → 浮动层出现（白色卡片 + 阴影）
  → 浮动层跟随手指移动（transform: translate3d GPU加速）
  → 根据移动距离判断目标组（>80px切换组）
  → 目标组显示 drag-over 高亮
  → 手指抬起 → 执行跨组移动或取消
```

### 8.2 拖拽状态

| 状态 | 说明 |
|------|------|
| 悬停中 | 手指按下不足1秒，不响应移动 |
| 就绪 | 悬停1秒后 dragReady = true |
| 拖拽中 | 移动超过20px，isDragging = true |
| 完成 | 手指抬起，执行移动或取消 |

### 8.3 交互说明

| 操作 | 效果 |
|------|------|
| 长按1秒后向上拖拽 >80px | 设备移到上一个组 |
| 长按1秒后向下拖拽 >80px | 设备移到下一个组 |
| 长按1秒后移动不足80px | 设备留在原组 |
| 快速点击/滑动 | 不触发拖拽，视为普通点击 |

### 8.4 视觉反馈

| 元素 | 效果 |
|------|------|
| 原始卡片 | opacity: 0.3 半透明 |
| 浮动层 | 白色卡片，阴影，跟随手指 |
| 目标组 | drag-over 高亮样式 |

### 9.3 交互流程

```
用户点击"添加设备"
  → 弹出添加设备弹窗（底部弹出）
  → 显示"📡 手动搜索"和"添加测试设备"按钮
  → 点击"📡 手动搜索"
  → 初始化蓝牙适配器（uni.openBluetoothAdapter）
  → 开始蓝牙扫描（uni.startBluetoothDevicesDiscovery）
  → 显示扫描动画（脉冲波纹）
  → 实时发现设备（uni.onBluetoothDeviceFound）
  → 5秒后自动停止扫描
  → 显示设备列表或"未发现设备"
  → 点击设备 → 添加设备到"未分组设备"
  → 弹窗关闭，首页更新
```

## 10. 测试预警功能

### 10.1 布局

```
┌─ 🚨 测试预警 ──────────────────────────────┐
│  选择一个设备模拟异常上报，用于测试预警功能  │
│         [📡 模拟设备异常]                   │
└──────────────────────────────────────────────┘
```

### 10.2 交互流程

```
点击"测试预警"区域
  → 弹出设备选择列表（所有设备 + "取消"）
  → 选择目标设备
  → 弹出异常类型选择列表（6种异常 + "取消"）
  → 选择异常类型
  → 触发模拟警报
  → 自动跳转到控制页面
  → 控制页面显示警报卡片 + 持续震动
```

### 10.3 模拟异常类型

| 指令 | 异常类型 | 说明 |
|------|---------|------|
| >AE1 | 机芯超温 | 机芯温度超过安全阈值 |
| >AE2 | 线盘超温 | 线盘温度过高 |
| >AE3 | 过流保护 | 电流超过额定值 |
| >AE4 | 电流异常 | 电流波形异常 |
| >AE5 | 锅具异常 | 未检测到锅具 |
| >AE6 | 线盘测温开路 | 温度传感器开路 |

### 10.4 数据传递

```
首页触发警报 → uni.setStorageSync('pendingAlarm', alarmData)
             → uni.$emit('deviceAlarm', alarmData)
             → uni.navigateTo('/pages/device/detail')
控制页面接收 → onShow 中检查 pendingAlarm
             → uni.$on('deviceAlarm') 监听事件
             → 显示警报卡片 + 启动持续震动
```
